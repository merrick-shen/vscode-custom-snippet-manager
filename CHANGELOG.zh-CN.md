# 更新日志

"Custom Snippet Manager" 扩展的所有重要变更都将记录在此文件中。

本文件遵循 [Keep a Changelog](http://keepachangelog.com/) 规范。

[English Changelog](./CHANGELOG.md)

## [未发布]

### 新增

- **文件夹**：新增文件夹功能，可创建、重命名、删除文件夹，对代码片段分类管理；新建或编辑片段时可选择所属文件夹
- **按文件夹导出**：导出时可勾选多个文件夹，每个文件夹单独导出为一个 JSON 文件
- **导入存放方式**：导入时可选择新建文件夹存放，或导入到已有文件夹

### 变更

- **图标系统**：所有内联 SVG 图标统一替换为 @iconify/vue 图标库（carbon 图标集）
- **侧边栏布局**：侧边栏改为按文件夹分组的可折叠列表
- **存储结构**：片段改为按文件夹分文件存储，旧数据自动迁移并保留原数据备份
- **导出文件**：导出文件现在按文件夹组织，文件名包含文件夹名称
- **重复检测**：导入到已有文件夹时，重复检测只针对该文件夹内的片段
- **导入兼容性**：1.2.0 及更早版本导出的文件（片段平铺数组格式）无法导入，仅支持新的文件夹格式

### 修复

- **保存按钮**：修复保存按钮在后端响应前恢复可点击状态的问题，避免极端情况下重复创建片段
- **语言标签**：修复适用范围为"全部"时语言标签始终显示"ALL"而不随界面语言切换的问题
- **语言选择器**：修复选择"全部"时下拉菜单关闭状态仅显示图标无文字的问题，现在同时显示图标和标签

## [1.2.0] - 2026-05-26

### 新增

- **设置入口**：侧边栏新增设置按钮，可查看插件版本、开源协议、源代码地址等信息
- **打开目录**：设置页面新增"打开代码片段目录"按钮，可直接在系统文件管理器中打开数据存储位置，同时显示存储路径
- **片段预览**：鼠标悬停在列表项上时弹出预览卡片，显示代码内容（带语法高亮）和描述，无需打开编辑即可快速查看片段内容
- **清空数据**：设置页面新增"清空所有数据"按钮，支持一键清除所有片段，带二次确认防止误操作

### 变更

- **通知弹窗位置**：侧边栏通知提示从页面顶部移至底部显示，避免弹出时挤压顶部内容布局
- **性能优化**：优化数据存储方式，避免片段较多时编辑器卡顿
- **性能优化**：使用片段补全时减少不必要的文件写入，提升高频使用时的流畅度
- **语言筛选多选**：侧边栏语言筛选支持同时选择多个语言，与编辑器的适用范围选择一致
- **架构优化**：统一语言配置数据源
- **语言切换按钮**：将翻译图标替换为当前语言的国旗标识，下拉菜单中每个语言也显示对应国旗
- **样式架构**：使用 Sass 重构前端样式代码，提升可维护性和可扩展性
- **排序偏好**：侧边栏排序方向现在会跨会话保持，不再每次打开侧边栏都重置

## [1.1.3] - 2026-05-25

### 新增

- **语言切换图标**：侧边栏语言切换按钮添加翻译图标，更直观地提示用户可切换插件语言

### 修复

- **导入ID覆盖**：修复导入片段时旧ID覆盖新生成ID的问题，合并策略下不再产生重复ID，非重复片段也能正确获得新ID
- **导入重复检测**：修复重复检测仅按 ID 匹配的问题，现在同时按名称+前缀匹配，重复导入同一文件时能正确弹出重复策略选择对话框
- **编辑器面板复用**：修复从编辑模式切换到新建模式时表单不清空的问题
- **编辑器面板标题**：修复编辑模式下面板标题始终显示"新建"的问题
- **语言字段匹配**：修复语言字段含空格时筛选和补全失效的问题，现在会自动去除多余空格
- **导入挂起防护**：添加超时和清理机制，防止关闭侧边栏后导入流程永久卡住
- **编辑器面板语言同步**：修复编辑器面板打开时切换语言不生效的问题，现在语言设置会实时同步到已打开的编辑器面板
- **删除通知时机**：修复删除片段时成功通知提前显示的问题，现在只有后端确认删除成功后才显示通知
- **编辑器错误残留**：修复切换编辑片段时上一次的错误提示不消失的问题

## [1.1.2] - 2026-05-24

### 修复

- **导出版本号**：修复 `importExportService.ts` 中 `APP_VERSION` 硬编码为 `'1.0.0'` 的问题，改为从 `package.json` 动态读取版本号，确保导出文件中的 `appVersion` 字段始终准确
- **C/C++补全**：修复 `LANGUAGE_ALIASES` 中 `cpp` 和 `c` 映射重叠的问题，C++片段不再在C文件中错误触发补全
- **数据安全**：修复数据文件损坏时静默重置为空数组的问题，现在会自动备份损坏文件并提示用户
- **导入性能**：修复导入片段时逐条写文件的问题，改为批量操作只写一次文件

## [1.1.1] - 2026-05-24

### 修复

- **包体积**：修复 `.vscodeignore` 未排除 `webview/node_modules/` 的问题，VSIX 从 ~29 MB 减小至 ~3.9 MB
- **包清理**：从打包文件中排除 webview 源码、构建脚本和配置文件

## [1.1.0] - 2026-05-24

### 新增

- **多语言选择**：片段适用语言字段支持多选
- **WebView 对话框**：使用自定义 WebView 对话框替换所有 VS Code 原生对话框（导入/导出确认、重复处理），与插件 UI 风格统一
- **WebView 通知**：使用自定义 WebView 通知替换 VS Code 原生通知，保持一致的 UI 体验
- **操作反馈**：新增片段创建、编辑和删除的成功提示通知
- **扩展国际化**：新增 **Русский / Deutsch / Français / Español / Português / Italiano / Polski / Türkçe** 支持（现已支持 13 种语言）

### 变更

- **国际化**：完善多语言支持——切换语言后所有界面文本均可正确切换
- **导入/导出**：导入导出结果提示改用 WebView 通知，并支持完整的 i18n 本地化

### 移除

- **冗余代码**：移除 SnippetService 中未使用的变更通知系统
- **冗余 i18n 键**：清理所有语言文件中未使用的验证和重复消息键
- **内部导出**：移除 ImportExportService 中内部接口不必要的 `export` 修饰符

## [1.0.0] - 2026-05-01

### 新增

- **片段管理**：支持创建、编辑和删除代码片段，包含名称、触发前缀、代码内容、描述和适用语言字段
- **智能补全**：基于 CompletionItemProvider 的前缀触发补全，支持 SnippetString 制表位光标跳转（`$1`、`$2`、`$0`）
- **侧边栏界面**：基于 Webview 的片段列表，支持模糊搜索（Fuse.js）和语言筛选
- **编辑器界面**：基于 Webview 的表单，支持 CodeMirror 6 语法高亮和输入校验
- **语言图标**：基于 Iconify 的语言图标，显示在下拉菜单和片段列表项中
- **国际化**：完整的简体中文 / 繁體中文 / English / 日本語 / 한국어 界面支持，下拉菜单切换语言，偏好跨会话持久保存
- **导入与导出**：导出代码片段为 JSON 文件（`code_snippet_config_YYYYMMDD_HHMMSS.json`），包含版本元数据；从 JSON 文件导入，支持数据验证、安全检查和重复处理（覆盖/跳过/合并）
- **排序功能**：按添加日期排序，支持正序/倒序切换；排序相关字段在导出时自动剔除
- **命令**：
  - `新建片段` — 打开编辑器创建新片段
  - `打开片段库` — 聚焦侧边栏面板
  - `插入片段` — 基于 QuickPick 的片段插入（`Ctrl+Alt+I`）
  - `触发片段补全` — 触发自动补全建议（`Ctrl+Alt+Space`）
  - `保存到片段库` — 通过右键菜单将选中代码保存为片段（自动识别语言）
- **右键菜单**：编辑器右键菜单中添加"插入片段"和"保存到片段库"选项
- **数据存储**：使用 VS Code 的 `globalStorageUri` 进行本地 JSON 文件持久化
- **删除确认**：自定义弹窗对话框，防止误删
- **错误处理**：侧边栏和编辑器视图中带有自动消失动画的错误提示条
